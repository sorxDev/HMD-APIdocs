API Reference
The howmanydrifters.com REST API gives you programmatic access to live Orion Drift statistics, player counts, fleet activity, match history, 3D position data, and news. All endpoints share the same base URL: https://howmanydrifters.com/api/v1 Every response is JSON. Successful responses include "success": true; failed ones include "success": false and an "error" key. Data refreshes approximately every 60 seconds.
Authentication
Pass your API token as a Bearer credential in the Authorization header on every request. You can generate a token from your account page after signing in with Discord. Authorization: Bearer hmd_<your_token> Endpoints labelled public can be called without a token. Endpoints labelled auth will return an error if the header is missing or invalid.
Errors
All error responses follow the same shape: success: false, a machine-readable error key, and a message string you can show to users. {"success": false, "error": "missing_range", "message": "start and end query params are required (unix seconds)"}
Public Endpoints
GET /api/v1/public/summary public ▶
Returns a live snapshot of the entire Orion Drift server, how many players are online right now, how many stations and fleets are active, and ranked top-10 lists for both fleets and stations by current player count. No auth required; safe to poll from a public dashboard.
Response Fields
FieldTypeDescription
active_stationsnumberNumber of game server stations currently running
active_fleetsnumberNumber of fleets with at least one player online
total_playersnumberTotal players in-game across all fleets right now
top_fleetsarrayTop fleets by player count, each has name, players, stations
top_stationsarrayTop stations by player count, each has name, region, players
Example Response
{
  "success": true,
  "active_stations": 49,
  "active_fleets": 15,
  "total_players": 1365,
  "top_fleets": [
    { "name": "Rookies", "players": 837, "stations": 18 },
    { "name": "Core",    "players": 195, "stations": 5  },
    ...
  ],
  "top_stations": [
    { "name": "EA Calibration_2", "region": "us-east-2", "players": 74 },
    ...
  ]
}
GET /api/v1/public/history public ▶
Returns the global playercount time-series for the past 7 days as an array of {ts, val} pairs, Unix timestamp and concurrent player count. Use this to draw charts or spot peak-hour patterns. Roughly 336 data points are returned per 7-day window.
Example Response
{
  "success": true,
  "global": [
    { "ts": 1781284800, "val": 1450 },
    { "ts": 1781289600, "val": 1823 },
    ...
  ]
}
GET /api/v1/public/users public ▶
Check whether the public user leaderboard is currently active. When enabled is false, individual player data is not exposed through the API.
Example Response
{
  "success": true,
  "enabled": false
}
Auth Endpoints
GET /api/v1/auth/me auth ▶
Resolves your Bearer token to the Discord account it belongs to. Returns the Discord user ID, username, and avatar hash. You can build the avatar image URL from these values using the Discord CDN pattern shown below.
Example Response
{
  "success": true,
  "user": {
    "id":       "1445774763409670164",
    "username": "sorxyzz",
    "avatar":   "9c0b3cf12b2beef81055d5b93e1afcde"
  }
}

// Avatar URL: https://cdn.discordapp.com/avatars/{id}/{avatar}.png?size=64
GET /api/v1/auth/permissions auth ▶
Returns the access level and feature flags granted to your token. The tier field tells you your account level. Each flag in flags controls whether a specific category of data is accessible, check these before calling gated endpoints to avoid unnecessary 403 errors.
Response Fields
FieldTypeDescription
tierstringYour account tier: "public" | "auth" | higher admin tiers
flags.can_view_chartsbooleanWhether you can query per-fleet playercount history
flags.can_view_matchesbooleanWhether you can query match and event statistics
flags.can_view_positionbooleanWhether you can query 3D player position track data
flags.can_view_adminbooleanWhether you have access to admin-only endpoints
Example Response
{
  "success": true,
  "tier": "auth",
  "flags": {
    "can_view_charts":   true,
    "can_view_matches":  true,
    "can_view_position": true,
    "can_view_admin":    false
  }
}
User Endpoints
GET /api/v1/customisation auth ▶
Returns your saved UI customisation, the active color palette with a full CSS variable map you can apply directly, your heatmap gradient and source palette settings, and a banner message shown to users on lower access tiers.
Example Response
{
  "success": true,
  "heatmap_gradient":       "plasma",
  "heatmap_source_palette": "default",
  "active_palette": {
    "id":    "original",
    "name":  "Original",
    "quote": "The classic SteamDB-style dark theme.",
    "colors": {
      "--bg":     "#161920",
      "--accent": "#4b9bf3",
      "--ok":     "#a3ec5d",
      ...
    }
  },
  "public_tier_banner": "To view per-fleet data..."
}
GET /api/v1/user/preferences auth ▶
Returns your saved user preferences as a key/value object. Used to persist UI choices across sessions, things like active filters, panel states, or view mode. Returns an empty object if no preferences have been saved yet.
Example Response
{
  "success": true,
  "preferences": {}
}
Charts Endpoints
GET /api/v1/charts/list auth ▶
Returns every active fleet with its current player count, station count, and UUID. Call this first to get the id values you need when querying /charts/history. UUIDs are stable, meaning they don't change if a fleet is renamed.
Example Response
{
  "success": true,
  "fleets": [
    {
      "id":        "e696ee24-e6fb-45ce-a6bf-e225d15ca9b9",
      "name":      "Rookies",
      "players":  843,
      "stations": 18
    },
    ...
  ]
}
GET /api/v1/charts/history auth ▶
Returns the playercount time-series for a single fleet over a time range you specify. Each data point is a {ts, val} pair. All three parameters are required, omitting start or end returns HTTP 400 with error: "missing_range". Maximum range: 7 days.
Query Parameters
ParameterTypeRequiredDescription
fleetstring (UUID)RequiredThe fleet UUID from /charts/list
startintegerRequiredStart of the time range, Unix timestamp in seconds
endintegerRequiredEnd of the time range, Unix timestamp in seconds
Example Response
{
  "success": true,
  "points": [
    { "ts": 1781281800, "val": 839 },
    { "ts": 1781282400, "val": 811 },
    ...
  ]
}
Match Endpoints
GET /api/v1/events/stats auth ▶
Returns match data for the past 7 days across all fleets. Includes a breakdown of matches played per arena district (Driftball, Club, Scraprun, etc.) and a full chronological list of every match with its start time, duration, player count, arena name, final scores, and the station and fleet it took place on.
Response Fields
FieldTypeDescription
tierstringScope level of the data, e.g. "global"
scopeobjectDescribes the filter: type, fleet_key, fleet_name
rangeobjectThe time window: start and end as Unix seconds
districtsobjectMatch counts keyed by arena district name
matchesarrayFull list of individual match records for the period
Match Object
{
  "started_ts":    1781285175,
  "stopped_ts":    1781287457,
  "duration_sec":  2282,
  "player_count":  7,
  "arena":          "Beta 02",
  "score_values":  [1, 2],
  "station_name":  "ODC_Matches_2",
  "station_region":"eu-central-1",
  "fleet_name":    "ODC_Matches"
}

// districts example:
"districts": { "Driftball": 6425, "Club": 1283, "Space": 2920, ... }
News Endpoints
GET /api/v1/news/config auth ▶
Returns the configuration for all four news panels, their display label and whether each is currently visible. The panels are official (studio announcements), community (fleet posts), competitive (league news), and developer (punda's blog). Check this before rendering the news feed.
Example Response
{
  "panels": {
    "official":    { "label": "Another Axiom Announcements",  "visible": true  },
    "community":   { "label": "Orion Drift Community Fleets", "visible": false },
    "competitive": { "label": "Competitive Events",            "visible": true  },
    "developer":   { "label": "Punda- News",                   "visible": false }
  },
  "competitiveSummaryEnabled": true
}
GET /api/v1/news/feed auth ▶
Returns all news items from enabled panels as a flat array. Each item has an id, a source tag matching the panel it comes from, a title, and content_html with sanitized HTML you can render directly.
Example Response
{
  "success": true,
  "items": [
    {
      "id":           1870,
      "source":       "community",  // official | community | competitive | developer
      "title":        "Minigame Monday",
      "content_html": "<h1>...sanitized HTML...</h1>"
    },
    ...
  ]
}
GET /api/v1/news/competitive/summary auth ▶
Returns a structured summary of all active and recently completed competitive leagues. Each entry has the league ID, name, region, current status, and a payload object with full standings, team data, and match results.
Example Response
{
  "success": true,
  "leagues": [
    {
      "league_id": "69ef997a2199c975c066f5bf",
      "name":      "Season 1 EU",
      "region":   "EU",
      "status":   "in_progress",
      "payload":  { ... }
    }
  ]
}
Position / Heatmap Endpoints
GET /api/v1/positions/fleets/events auth ▶
Returns the list of fleets with position tracking enabled for the events level. Each entry has a short key identifier, a name, and a full UUID. max_range_secs tells you the maximum query window, 7 days.
Example Response
{
  "success":        true,
  "level_type":     "events",
  "originVisible":  true,
  "max_range_secs": 604800,
  "fleets": [
    { "key": "CH",   "name": "Clubhouse",      "id": "7085995a-..." },
    { "key": "COMP", "name": "Scrims",         "id": "debdecb5-..." },
    { "key": "ODC",  "name": "ODC_Tournaments", "id": "bd6946d4-..." },
    { "key": "RF",   "name": "Rookies",        "id": "e696ee24-..." },
    { "key": "STRK", "name": "Strike",         "id": "6c9d1fe1-..." }
  ]
}
GET /api/v1/positions/fleets/orion auth ▶
Same shape as the events fleet list, but scoped to the orion map level. Fewer fleets are available here. Use the key from this response when querying /positions/orion.
Example Response
{
  "success":        true,
  "level_type":     "orion",
  "originVisible":  true,
  "max_range_secs": 604800,
  "fleets": [
    { "key": "CH",   "name": "Clubhouse", "id": "7085995a-..." },
    { "key": "RF",   "name": "Rookies",   "id": "e696ee24-..." },
    { "key": "STRK", "name": "Strike",    "id": "6c9d1fe1-..." }
  ]
}
GET /api/v1/positions/orion auth ▶
Returns 3D movement tracks for players in the Orion over a given time range. Each track belongs to a fleet and contains [x, y, z, timestamp] points in world-space coordinates. Responses can be 700 KB or more for busy periods. If truncated is true, narrow your time range and retry.
Query Parameters
ParameterTypeRequiredDescription
startintegerRequiredStart of the time range, Unix timestamp in seconds
endintegerRequiredEnd of the time range, Unix timestamp in seconds
includePathsbooleanOptionalPass true to include full movement path arrays. Omit for point-only data, which is much smaller.
Example Response
{
  "success":    true,
  "level_type": "orion",
  "truncated":  false,
  "warnings":   [],
  "fleets":     [...],
  "tracks": [
    {
      "fleetIdx": 0,
      "points": [
        [2559.5, 18286.6, -29089.9, 1781883032],
        [2154.2, 20339.5, -29335.9, 1781883152],
        // [x, y, z, unix_timestamp]
      ]
    }
  ]
}
